# Job Recommender System
## Описание ТЗ:
Разработка простого симулятора системы рекомендаций вакансий - Job Recommender. <br>
В системе можно создать профиль пользователя (опыт, технологии, интересы) <br>
и вакансии(теги, описание). На основе совпадений, система предлагает наиболее релевантные <br>
вакансии для пользователей. <br>

На первом этапе необходимо создать скелет работающего приложения. Создать сущности, <br>
с которыми будем работать. А также простой CLI (Command Line Interface) интерфейс для <br>
работы с ними. Названия сущностей, их набор можно назвать по своему. Ниже пример: <br>

 - Создать основные классы:
   - для пользователей системы (имя, список навыков, опыт);
   - для вакансий, которые будут публиковаться в системе (названия, компания, теги, 
   требуемый опыт) <br>
   - для мэтча между пользователями и вакансией (результат сопоставления - насколько <br>
   мэтчится пользователь с вакансией). <br>

 - Реализовать CLI-команды: <br>
   - user <name> --skills=... --exp=.... Добавляет пользователя в систему. <br> 
   Например: <br>
   ```bash
      user alice --skills=java,ml,linux --exp=2
   ```
   - user-list — выводит список пользователей (каждого с новой строки) в формате: <br>
   ```text
      Alice java,ml,linux 2 
      Bob c++,python 5
   ```
   - job <title> --company=... --tags=... --exp=... Добавляет вакансию. <br>
  Например:
    ```bash
    job Backend_Dev --company=VK --tags=java,backend,linux --exp=1 <br>
    ```
    - job-list — выводит список вакансий (каждую с новой строки) в формате: <br>
    ```text
    Backend_Dev at VK 
    Java_developer at Google
    ```
    - suggest <username> — выводит не больше 2 вакансий, подходящих указанному пользователю. <br> 
    Степень того, насколько подходит вакансия предлагается вычислять как количество совпадений <br> 
    между skills юзера и tags вакансии. Если опыта (exp) у юзера не хватает, то поделить <br>
    полученное количество пополам. <br>
    - exit — завершает работу приложения с кодом 0. System.exit(0); <br>
 - Хранить данные в памяти <br>
 - Для реализации CLI использовать **java.util.Scanner** <br>

### Формат входных данных
На вход подается любой набор CLI команд из описания. Например: <br> 
```bash
user alice --skills=java,ml,linux --exp=2
```
Список параметров команды не сортирован <br>
```bash
user bob --exp=10 --skills=java,java,java
```
Каждый набор входных данных заканчивается командой **exit**. <br>

### Формат выходных данных
Выходные данные в ответ на команду **user-list**
— в формате <name> <skills> <exp>
```text
alice java,ml,linux 2
```
Выходные данные в ответ на команду **job-list** — в формате <job_title> at <company>
```text
Backend_Dev at VK 
Java_developer at Google
```
Формат выходных данных в ответ на команду **suggest** такой же, как и на команду job-list

### Пример 1
Входные данные:
```bash
user alice --skills=java,ml,linux --exp=2
user bob --skills=java,java,java --exp=10
user-list
exit
```
Выходные данные:
```text
alice java,linux,ml 2
bob java 10
```

### Замечание
- Старайтесь реализовывать функционал по принципу Tell Don't Ask (из лекции). <br>
- Понятная структура программы, названия переменных, методов вам самим поможет <br> 
дальше работать с программой. <br>
- Повторное создание юзера с тем же именем, или вакансии с тем же названием не должно <br>
приводить к перезаписи ранее созданной сущности. <br>
- Не стоит выводить в консоль посторонние символы. <br>

## 💬 Пример сессии
```bash
# Добавляем пользователей
user alice --skills=java,ml,linux --exp=2
user bob --skills=java,java,java --exp=10

# Просмотр пользователей
user-list
# Вывод:
# alice java,linux,ml 2
# bob java 10

# Добавляем вакансии
job Backend_Dev --company=VK --tags=java,backend,linux --exp=1
job ML_Engineer --company=Google --tags=ml,python --exp=3

# Просмотр вакансий
job-list
# Вывод:
# Backend_Dev at VK
# ML_Engineer at Google

# Получаем рекомендации для alice
suggest alice
# Вывод (топ-2 по релевантности):
# Backend_Dev at VK        # score=2.0 (2 совпадения, опыт хватает)
# ML_Engineer at Google    # score=0.5 (1 совпадение, опыт не хватает → /2)

# Завершение
exit
```
